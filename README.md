## Versioning

Each update to any document in this repository is tagged using the following naming convention:

```
{product}-v{major}.{minor}
```

- A major version change indicates that the terms have changed in a way that affects the contract.
- A minor version change is for typo fixes or other non-substantive edits that do not affect the contract.

**Examples:**
- `inbox-academy-v1.0` — Initial release of InboxAcademy contracts.
- `inbox-academy-v2.0` — Major update to one or more InboxAcademy contracts.

Tags allow any version ever referenced in our products to be retrieved at any time.
